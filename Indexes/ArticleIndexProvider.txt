store.RegisterIndexes<ArticleIndexProvider>(); ne produit pas directement de résultat affichable.
➡️ C’est une étape de configuration, pas une opération qui retourne des données.

Mais tu peux bien sûr vérifier que les index ont été créés et voir leurs effets
(ce qu’ils contiennent, les résultats produits par l’index ArticleByWord, etc.).
Je vais t’expliquer exactement comment afficher les résultats concrets de ce registre d’index.

1️/ Que fait vraiment cette ligne :
store.RegisterIndexes<ArticleIndexProvider>();
Elle déclare au moteur YesSql :

“Quand un Article est ajouté ou modifié, applique la logique du ArticleIndexProvider 
pour mettre à jour les index.”

Donc elle ne renvoie rien (c’est un void).

Elle prépare YesSql à construire les tables et les index quand tu vas sauver des objets.

👉 Si tu veux “voir” un résultat, tu dois interroger les index qu’elle met à jour
(par exemple ArticleByWord).

2️/ Étapes pour afficher les résultats de l’index

Voici la méthode complète (en partant du code standard du projet YesSql FullText).
Étape 1 — Sauvegarder un article :

using (var session = store.CreateSession())
{
    await session.SaveAsync(new Article
    {
        Title = "Découverte de YesSql",
        Content = "YesSql est un moteur de base de données orienté documents."
    });
    await session.CommitAsync();
}

Cela déclenche automatiquement la construction de l’index ArticleByWord grâce au
ArticleIndexProvider.

Étape 2 — Interroger la table d’index

Ensuite, pour voir ce que l’index contient, tu fais :

using (var session = store.CreateSession())
{
    var results = await session.QueryIndex<ArticleByWord>().ListAsync();

    foreach (var r in results)
    {
        Console.WriteLine($"{r.Word} : {r.Count}");
    }
}

Cela va afficher tous les mots trouvés et leur nombre d’occurrences, par exemple :

YesSql : 2
moteur : 1
base : 1
données : 1
orienté : 1
documents : 1

Étape 3 — Facultatif : filtrer ou trier

Tu peux aussi interroger l’index pour chercher un mot spécifique :

var yesqlWord = await session.QueryIndex<ArticleByWord>(x => x.Word == "YesSql").FirstOrDefaultAsync();
Console.WriteLine($"{yesqlWord.Word} : {yesqlWord.Count}");

3️⃣ En résumé :
| Étape                                           | Action                             | Résultat                              |
| ----------------------------------------------- | ---------------------------------- | ------------------------------------- |
| `store.RegisterIndexes<ArticleIndexProvider>()` | Enregistre la logique d’indexation | ✅ Configuration prête                 |
| `session.SaveAsync(new Article(...))`           | Ajoute des données à indexer       | 🔄 Déclenche l’indexation             |
| `session.QueryIndex<ArticleByWord>()`           | Requêtes sur les index créés  
-----------------------------------------------------------------------------------
